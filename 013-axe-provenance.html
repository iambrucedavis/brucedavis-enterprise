01 — The Fleet six agents, six identities

Every agent is an identity.

Not a task queue — a fleet of distinct identities. Each holds a standing permission scope, a token budget, and an accountable owner. The interface makes all of it visible at a glance.

02 — The Gate the agent asks; the person decides

It asks before it acts.

Remediation needs to revoke a compromised credential. Its standing scope doesn't allow it. The system does not override the rule — it stops the agent and asks the operator.

INC-7731 · escalation [ BLOCKED — AWAITING OPERATOR ]
scope check  revoke_sessions requires elevated; Remediation holds read_write
escalation  Remediation requests elevated scope — operator approval required
  → operator approves  ·  Provisioner mints a time-boxed credential
  → Remediation acts  ·  sessions revoked, credentials rotated

The escalation isn't hardcoded to pause here. It's a real permission check failing inside the agent loop — the gate is the governance, not a scripted demo moment. Approve, and a credential is issued, used, and expires. Deny, and the agent holds at its standing scope.

03 — The Interface the UI renders the agents, not the reverse

The UI is generative.

The backend emits a structured event stream. The control room renders against that stream and re-renders as it changes — the agents are never hardcoded into the interface. That is the technical claim the project demonstrates.

Generative, not hardcoded
The control room is a projection of the backend's JSON state. Change the fleet on the backend and the UI renders the change — no interface edits.
Human-in-the-loop
Every privilege escalation pauses the fleet. Nothing elevates, and nothing destructive runs, until a person approves it.
One frontend, two transports
Demo mode replays a recorded run; live mode streams from the backend. Same UI, same reducer — proof the render is data-driven.
In review — what I'd do differently
  • The agents' decisions are deliberate and legible, not LLM-improvised — the demo is about the system being governable, not the agents being clever.
  • One incident scenario ships; the event-stream contract is built to carry many, and the next step is a second one.
  • The backend runs locally by design — the deployed demo replays a recorded run, so there is no key to leak and no per-visit cost to carry.
